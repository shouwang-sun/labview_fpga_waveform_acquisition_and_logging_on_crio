CompactRIO上的LabVIEW FPGA波形采集与记录

LabVIEW 2013

CompactRIO的LabVIEW FPGA波形采集和记录的范例项目采集连续波形数据并将数据记录到实时控制器的磁盘上。由LabVIEW FPGA模块创建的采集代码在CompactRIO设备内的FPGA上运行。

该范例项目是基于“简单状态机”和“队列消息处理器”模板。关于模板工作的详细信息，可通过新建项目对话框获取“简单状态机”和“队列消息处理器”模板及文档。

功能

  • 带有可选用户界面的无外设操作－用户界面VI与CompactRIO交互并显示数据。VI可随时连接或断开与设备的连接，且不会影响采集和记录循环。
  • 触发的数据记录－满足触发器条件时，实时VI将采集到的数据以TDMS文件格式记录到磁盘。范例项目同时管理使用的磁盘空间量。
  • 错误处理－应用报告并记录CompactRIO设备的全部错误。设备在遇到关键性错误时将关闭。

位于FPGA接口模式的NI CompactRIO设备

该范例项目专用于带有下列组件的NI cRIO-9074：

  • 软件：
    • LabVIEW Real-Time
    • 网络流
    • 网络变量引擎
    • NI系统配置
    • NI-RIO
    • NI-Watchdog
  • NI C系列加速计/麦克风输入模块。该范例项目专用于读取插槽1中的NI 9234模块的4个模拟输入通道。

概述

范例项目由跨3个执行终端的9个并行循环组成。下列循环在台式机上并行运行：

  • 处理用户界面事件（UI Main.vi - 事件处理循环)－根据前面板事件生成发送至“UI消息循环”的消息。
  • 处理用户界面和实时控制器信息（UI Main.vi - UI消息循环）－使用网络流接收和响应来自事件处理循环和RT消息处理循环的消息。
  • 显示来自CompactRIO设备的消息和数据（UI Main.vi - 监控循环）－显示RT循环采集到的信息的最新值－System Health and FPGA Monitoring.vi

下列循环在实时控制器上并行运行：

  • 处理来自用户界面的命令（RT循环- UI Commands.vi）－读取开发计算机上通过UI Main.vi发送的命令，并生成相应的消息。
  • 处理来自实时控制器的全部循环的消息（RT Main.vi - 消息处理循环）－处理运行在实时控制器上的所有循环的消息。
  • 确保RT控制器保持响应（RT循环－Watchdog.vi）－发送信号至看门狗，确保RT控制器保持响应。
  • 监视实时控制器的诊断信息和来自FPGA的数据（RT循环－System Health and FPGA Monitoring.vi）－监控实时控制器的CPU和内存使用量及来自FPGA VI的数据。该信息被写入网络发布的共享变量并出现在UI Main.vi的系统监控选项卡。
  • 采集和记录来自FPGA的数据（RT循环－Acquisition and Logging.vi）－运行FPGA Main.vi，读取DMA FIFO的数据并将数据记录至磁盘。

下列循环在FPGA上运行：

  • 采集数据（FPGA Main.vi－采集循环）－从C系列模块输入端采集数据并将数据写入至DMA FIFO。

运行范例项目

  1. 匹配范例项目至硬件。
  2. 在项目浏览器窗口，打开我的电脑»工具 - Configuration File Generator.vi
  3. 输入适用于用户应用程序的配置值。确保在TDMS属性数组中查看全部的值。
  4. 运行VI。LabVIEW生成配置文件Config.xml，文件与.lvproj位于同一目录。
  5. 将该XML放置在实时控制器的根目录文件夹。
  6. 运行RT CompactRIO终端» Main.vi。VI使用Config.xml文件中的配置设置采集和记录数据。
  7. 打开并运行我的电脑»UI Main.vi。
  8. 在控制器地址文本框中输入CompactRIO设备的IP地址，并单击连接。
  9. 完成连接后，单击采集实时数据在波形图表中显示实时数据。更改实时数据通道控件，查看不同通道的实时数据。
  10. 选择退出退出应用程序。

匹配范例项目至硬件

范例项目中的FPGA VI编译用于指定的FPGA和I/O硬件。如用户使用不同的FPGA或C系列模块，请按照下列步骤匹配范例项目至硬件。下列步骤适用于NI CompactRIO设备，单也可匹配范例项目至NI单板RIO设备。

  1. 确保全部设备均已配置并与开发计算机连至同一网络。
  2. 在项目浏览器窗口中，添加或找到RT CompactRIO终端至顶层项目项。
  3. 添加或找到用户在上一步骤中添加的RT Compact RIO终端的CompactRIO机箱。确保机箱被设置为LabVIEW FPGA接口模式。
  4. 添加或找到用户在上一步添加的CompactRIO机箱的FPGA终端。当提示部署设置时，单击以后部署。
  5. 添加或找到用户在上一步骤中添加的FPGA终端的C系列输入模块。
  6. 将默认RT CompactRIO终端中的下列项目项拖曳至在步骤2中添加的终端：
    • Error Handlers文件夹
    • Globals文件夹
    • Support VIs文件夹
    • RT Loops文件夹
    • Type Definitions文件夹
    • Shared Variables.lvlib
    • RT主程序
  7. 将默认FPGA终端中的下列项目项拖曳至在步骤4中添加的终端：
    • Support VIs文件夹
    • Type Definitions文件夹
    • FPGA主程序
  8. 删除不再具有关联VI的默认RT CompactRIO终端项目项。
  9. 重新建立RT主程序(RT Main.vi)与位文件间的链接：
    1. 打开RT CompactRIO终端的“Support VIs”文件夹内的Start Acquisition and Logging.vi。
    2. 从项目浏览器窗口拖曳FPGA Main.vi至“打开FPGA VI引用”函数。
  10. 打开FPGA Main.vi并确保FPGA I/O节点使用用户所需的输入通道。例如，需要FPGA I/O节点读取Mod2/AI5，而非Mod1/AI2。实际读取的通道取决于应用程序。

    默认情况下，VI中的FPGA I/O节点读取Mod1/AI0–3。

当前可编译FPGA VI且将其视作应用程序的一部分运行。

配置范例项目的设置

在项目浏览器窗口，打开我的电脑»Globals»Global - Configuration Options.vi并配置范例项目的设置。
